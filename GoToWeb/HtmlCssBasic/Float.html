Поток документа - очерёдность всех элементов документа, которые создаются сверху вниз каждый элемент стремится встать как можно выше, как можно левее. float: left - говорит элементу прижаться как можно левее, а все что идет после тебя будет обтекать тебя справа :right :none есть поток документа картирка->параграф когда картинке присваивается float - картинка как бы выпадает из потока (выдвигается на верхний слой), но текст параграфа сдвигается до размеров картинки парграф как-бы находится за картинкой - обтекаемость распространяется на все элементы, которые следуют за элементом с float - чтобы убрать обтекаемость для следующего элемента - clear: left, right, both; - когда блочным элемента присваивается float - то свойство растягвания на всю ширину родителя не работает, и фактически они становятся inline-block элементами т.е. ширина по контенту - если к строчному элементу применить float - то он становиться inline-block, т.е. к ниму применимы задание высоты и ширины - можжно провести аналигию свойств float == display: inline-block, но это не одно и тоже - для элементов с float - проблема выпадения из потока решается : - создание еще одного элемента со свойством clear:both; (class=clearfix) - с помощью псевдоэлемента :after - т.к. картинка является inline-block элементом - справа от неё может находиться другие элементы - с помощью float - можно делать сетки
